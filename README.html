Sped Delivery - Round 1 Task

Overview

This repository contains a backend implementation for Sped Delivery's Round 1 practical challenge. The task focuses on managing Orders, Delivery Zones, and Delivery Men with proper delivery assignment logic.

Features Implemented

1. Delivery Zones

  • Restaurants can define delivery areas in two ways: Polygon Zones and Radius Zones.
  • Each zone is linked to a restaurant and validated when placing orders.

2. Order Placement & Validation

  • Orders are accepted only if the delivery address falls inside the restaurant's delivery zone.
  • Distance between restaurant and customer is calculated using the Haversine formula.
  • Invalid addresses are rejected with a proper error message.

3. Delivery Men & Assignment

  • Tracks delivery personnel with name, phone, email, vehicle number, status, latitude, and longitude.
  • Assigns the nearest available delivery man within 5 km automatically.
  • Updates delivery man's status to busy once assigned.

4. Order Status Management

  • Delivery men can accept, reject, or update order status (processing, delivered, cancelled).
  • Orders show assigned delivery man and current status on the dashboard.

5. Admin Dashboard

  • Laravel Blade templates for Orders, Delivery Men, and Delivery Zones.
  • Sidebar navigation for quick access.
  • Distance and assignment info displayed clearly.

Technical Details

  • Framework: Laravel 12
  • Database: MySQL
  • Authentication: Laravel Sanctum (optional for API)
  • Distance Calculation: Haversine formula
  • Geo Logic: Custom PHP functions for radius and polygon checks

Key Models

  • Order: Tracks orders with assigned delivery man and delivery zone.
  • DeliveryMan: Tracks delivery personnel and status.
  • DeliveryZone: Stores polygon or radius-based zones for restaurants.
  • Restaurant: Stores basic restaurant info and delivery zones.

How to Setup

  1. Clone the repository: git clone <repository-link>
  2. Install dependencies: composer install && npm install && npm run dev
  3. Configure .env with database credentials.
  4. Run migrations: php artisan migrate --seed
  5. Serve the application: php artisan serve
  6. Access dashboard at http://127.0.0.1:8000

Assumptions & Design Choices

  • Each restaurant must have at least one delivery zone.
  • Delivery men are only assigned if available and within 5 km radius.
  • Order distances are calculated in kilometers.
  • No external spatial libraries were used; polygon and radius checks implemented in PHP.

Optional Improvements

  • Interactive map for creating delivery zones (Leaflet.js / Google Maps API).
  • Unit & feature tests using PHPUnit.
  • Notification system for delivery men.
